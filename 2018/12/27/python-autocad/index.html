使用Python读写DXF和DWG文件

发表于 2018-12-27

最近做了一点尝试，希望用Python生成dxf或者dwg格式文件，以便在Autocad软件中使用。Autocad 一直是二维绘图软件的霸主，这么些年无人能撼动他的地位，而Autodesk公司一直较为封闭，导致很少有开源的第三方库能很好的支持。

  • DWG: 脱离AutoCAD软件环境读取、写入DWG格式图纸文件,基本无法实现。通过COM自动化操作是一个可选的方案。
  • DXF: 它被AutoCAD作为与其它软件之间进行CAD数据交换的文件格式，有不少开源库支持。

根据需求，进行一番测试，以便选择合适的库进行开发，如果你正好也想做类似的工作，希望对你亦有用。测试包括两项内容：

  • 给定点集，自动绘制封闭的polyline
  • 对第一步产生的封闭区域，自动填充。

Python 生成DXF

有不少开源库支持DXF文件的读取和写入,一些用纯Python语言写的，比如：

  • dxf version 0.8.9
  • ezdxf
  • dxfgrabber

也有重量级的，如GDAL。纯Python语言实现的库，在功能上还相对单薄。

ezdxf

ezdxf文档还算不错，提供LWPoline 和 Hatch 函数，这正是我想要的。

dxfgrabber

dxfgrabber库支持的函数很多，绘图能力应该不错，但没有找到和填充有关的内容。

其他

其他的库，比如dxfwrite也大同小异，没有找到和填充有关的内容。

Pyhon 生成DWG

没有纯Python库支持，也没有开源的第三方库，好在AutoCAD支持COM。 用VB，c# 实现应该比较容易，但Python缺有点艰难。pyWin 和 comtypes 是两个可用的Pyton库，实现COM操作。但这两个库都久未更新，这条路也不是那么好走。 comtypes 看上去更新一点，用的也比较多，重点测试这个库。测试内容有两个，绘制封闭的Polyline，并对它进行填充。
